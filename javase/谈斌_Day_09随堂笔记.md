### Day_09随堂笔记

#### 猫和狗的案例

#### final

```java
final : 最终的 -> 阉割手术
    
能作用的位置:
	类: 最终的类,不能被继承 (没儿子)
	成员变量:最终的成员变量 -> 自定义常量
	    自定义常量标准的命名格式:
	        public static final 数据类型 常量名 = 初始值;
	        常量名: 必须全部大写,多个单词之间使用_分隔
	成员方法: 最终的成员方法 -> 不能被重写
	局部变量: 变成了局部范围内的常量 不能修改
	    实时final -> JDK7推出
	    当局部变量需要被final修饰的时候,final关键字会由JVM自动加上~
```

#### 抽象类

```java
抽象类: 比类更为抽象的类;

为什么会有抽象类:
	当父类中的方法不太好编写方法体去适应所有的子类,那么我们可以把父类中的该方法定义成抽象方法,而抽象方法必须存在于抽象类中,所以 有抽象类的概念
        
如何鉴别抽象类: 若一个类被abstract关键字修饰,那么就是一个抽象类        
    
abstract : 抽象的
    类: 抽象类
    方法: 抽象方法    
```

#### 抽象类的成员

```java
    1. 抽象类中可以什么都没有
    2. 成员变量 : 可以有
    3. 自定义常量 : 可以有
    4. 构造方法 : 可以有
    5. 普通成员方法 : 可以有
    6. 静态成员方法 : 可以有
    7. 抽象方法 : 可以有
        --> 如果一个类中没有抽象方法那么这个类也没有必要定义成抽象类
```

#### 抽象方法		

```java
抽象方法: 被abstract修饰的方法就是抽象方法;

特点:
	1. 没有方法体!
    2. 子类必须重写或者子类自己变成抽象类
        
格式:
	权限修饰符 abstract 返回值类型 方法名(形参列表);

抽象方法的出现:
	1. 抽象方法必须存在于抽象类(接口)中
    2. 子类继承抽象父类必须重写父类中所有的抽象方法或者子类自己变成抽象类
```

#### 抽象类的使用

```java
抽象类都是作为父类存在,因为抽象类中的抽象方法可以约束子类的行为.
    
抽象类不可以创建对象!!!!但是抽象类中可以有构造方法!!
    
1. 子类继承抽象父类,重写父类中所有的抽象方法
2. 子类继承抽象父类,把子类自己也变成抽象类,让子类的子类去实现抽象方法~    
```

#### 接口

```java
interface : 接口

为什么有接口:
	若一个抽象类中有且仅有抽象方法的时候,那么我们可以把这个抽象类定义成接口
        
接口的概念: 比抽象类更为抽象的类(接口是一种特殊的类)        
```

#### 接口的思想

```java
USB接口,API(应用软件开发接口)
    
接口: 是为了提供规范的!!!--> 好处: 方便
    
 规范要公开!!! -> public
```

#### 接口的定义格式

```java
格式:
	public interface 接口名{
        
    }

	接口名的命名方式和类名一致: 大驼峰!

类和类之间的关系: 继承 extends        
    单一继承,多层继承;
接口和类之间的关系: 实现 implements
    多实现  -> 接口的出现也打破了继承的单一性

一个类可以继承一个父类实现多个接口 ---> 格式:
	public class 子类 extends 父类 implements 父接口1,父接口2....{
        
    }
接口和接口之间的关系: 继承 extends
    多继承 :
    public interface 子接口 extends 父接口1,父接口2,...{
    
	}
```

#### 接口内部的成员

##### JDK8之前的接口 -> 最常用

```java
	1. 接口中可以什么都没有
    2. 成员变量 : 不可以有
    3. 自定义常量 : 可以有
        而且接口中所有的成员变量都是自定义常量,默认被public static final所修饰
    4. 构造方法 : 不可以有 -> 接口不可以创建对象
    5. 普通成员方法 : 不可以有
    6. 抽象方法 : 可以有
         接口中的方法都是抽象方法(JDK8之前是对的)
         接口中的方法默认被 public abstract 所修饰

      接口的思想: 接口是对外提供规范的,规范要公开!
```

##### JDK8的接口

```java
	JDK8基于JDK7,新增了2个成员:
    1. 默认方法
        格式:
            public default 返回值类型 方法名(形参列表){
                方法体;
            }
         默认方法默认被public修饰
    2. 静态方法
        格式:
            public static 返回值类型 方法名(形参列表){
                方法体;
            }
         静态方法默认被public修饰
```

##### JDK9的接口

```java
JDK9基于JDK8,新增了2个成员:
     1. 普通私有方法 : 是为默认方法服务的
        格式:
            private 返回值类型 方法名(形参列表){
                方法体;
            }
     2. 静态私有方法 : 是为静态方法服务的
        格式:
            private static 返回值类型 方法名(形参列表){
                方法体;
            }
```

#### 各版本接口使用步骤和注意事项

```java
接口使用的步骤:
        1. 创建一个类实现接口 -> 创建的类叫实现类
        2. 一般情况都是把接口中所有的抽象方法进行重写(实现)
        3. 在测试类创建实现类对象,对象调方法

    注意事项:
        1. 当父接口中有默认方法,实现类中可以选择性的重写父接口中的默认方法
        2. 接口中的默认方法可以使用实现类对象直接调用
        3. 多个父接口中有相同的默认方法,那么实现类中必须重写父接口中的默认方法
        4. 接口中的静态方法只能用接口的名字直接调用
        5. 接口中的私有方法只能在接口内部使用
```

#### 接口的好处

```java
1.  从父接口往实现类看:
		父接口约束了子类的行为
2. 从实现类往父接口看:
		父接口拓展了子类的行为            
```
